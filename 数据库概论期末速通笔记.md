## 二、各章核心知识点

### 第 1 章 数据库系统简介

#### 1. 数据独立性（PDF 第18页）
- **定义**：当数据的结构发生变化时，通过系统提供的映象（转换）功能，使应用程序不必改变。
- **物理独立性**：数据**存储结构**变化时，应用程序不变。
- **逻辑独立性**：数据**逻辑结构**变化时，应用程序不变。
- **实现视角**：将数据管理从应用程序中剥离出来。

#### 2. 文件系统在数据管理方面的不足（PDF 第29页）
1. **数据与程序紧密结合**：逻辑结构改变必须改程序。
2. **数据分散管理**：多文件、多格式，信息孤岛。
3. **数据语义只能由程序解释**：共享数据需了解对方语义。
4. **数据冗余、不一致**：多副本难维护。
5. **完整性难于维护**：约束需程序 DIY。
6. **查询困难**：记录之间无联系，每个查询都需重新编码。
7. **根源**：文件系统是无结构的。

#### 3. 数据库系统在数据管理方面的特性（PDF 第40页）
- 数据结构化、统一存取、维护数据语义与结构。
- **高数据独立性**：用户只需提出“做什么”，无须说明“怎么做”。
- 数据共享性高、冗余度低、一致性好。
- 数据由 DBMS 统一管理和控制（完整性、安全性、并发、恢复）。

#### 4. 数据模型（PDF 第50页）
- **定义**：数据库系统中用于提供信息表示和操作手段的形式构架。
- **两类**：
  - **概念数据模型**：按用户观点建模（ER 模型、UML/ODL），用于交流。
  - **结构数据模型**：按计算机实现观点建模（层次、网状、关系、对象）。
- **结构数据模型三要素**：
  1. **数据结构**：描述静态特性（对象类型、联系）。
  2. **数据操作**：检索、更新，定义操作含义/符号/规则/语言。
  3. **数据约束条件**：完整性规则集合。
- **历史模型比较**：层次模型（树）、网状模型（图）、关系模型（表）、对象模型；核心问题是“如何表达联系”。

#### 5. 数据库模式：三级模式、两级映像（PDF 第80页）
- **三级模式**：
  - **外模式（Sub-Schema）**：用户数据视图，局部逻辑结构，模式的子集。
  - **模式（Schema）**：全体数据的公共视图，全局逻辑结构。
  - **内模式（Storage Schema）**：存储模式，物理结构及存储方式（索引、压缩、列式等）。
- **两级映像**：
  - **外模式/模式映像**：保证**逻辑独立性**。模式改变时改映像，外模式不变。
  - **模式/内模式映像**：保证**物理独立性**。存储结构改变时改映像，模式不变。

#### 6. 数据库如何保证数据独立性（PDF 第83页）
- 通过三级模式结构 + 两级映像：
  - 物理独立性靠模式/内模式映像。
  - 逻辑独立性靠外模式/模式映像。

#### 7. DBMS 功能（PDF 第95页）
- **数据定义功能**：DDL 描述外模式、模式、内模式，经模式翻译程序存入数据字典。
- **数据存取功能**：DML（宿主型 / 自含型）实现检索、插入、修改、删除。
- **数据管理功能**：
  - 数据库运行管理：并发控制、存取控制、完整性检查、日志、事务管理、自动恢复。
  - 数据组织存储和管理：用户数据、索引、数据字典的组织存储。
  - 数据库建立和维护：装入、转换、卸出、转储、恢复、性能监视。

#### 8. DBA 职责（PDF 第100页）
- **建库**：确定模式、外模式、存储结构、存取策略；数据整理和装入。
- **用库**：定义完整性约束、保密级别、用户权限；监督控制运行；制定恢复策略。
- **改进**：性能分析、数据库重组织 / 重构造。

### 第 2 章 实体-联系模型（ER 模型）

#### 1. 基本概念（PDF 第3页）
- **实体（Entity）**：可区别于其他对象的客观存在或概念。
- **联系（Relationship）**：实体之间的关联。
- **属性（Attribute）**：实体的特征。
  - 简单属性 / 复合属性
  - 单值属性 / 多值属性
  - 派生属性
- **码**：
  - **超码（Superkey）**：能唯一标识实体的属性或属性组；超码的任意超集也是超码。
  - **候选码（Candidate Key）**：任意真子集都不能成为超码的最小超码。
  - **主码（Primary Key）**：从候选码中选定一个用于区别实体。
- **联系的码**：
  - M:N 联系：参与实体的主码共同组成。
  - 1:N 联系：多方实体的主码。
  - 1:1 联系：视参与情况选择一方主码。

#### 2. 联系的种类、势（PDF 第36页）
- 一对一（1:1）、一对多（1:N）、多对多（M:N）。
- 参与：全部参与（双线）/ 部分参与（单线）。

#### 3. 弱实体（PDF 第69页）
- **弱实体集**：所有属性都不足以形成主码的实体集。
- **标识性联系**：弱实体与其拥有者（强实体）之间的联系，双边框菱形。
- **分辨符（Discriminator / 部分码）**：弱实体集中区别依赖于某个强实体的属性集合，用下划虚线标明。
- **弱实体主码 = 强实体主码 + 弱实体分辨符**。
- 何时引入：弱实体参与其他联系，或本身含很多属性；层次结构一部分；多值/复合属性可抽成弱实体。

#### 4. 特化、概化、聚集（PDF 第61页）
- **特化（Specialization）**：自顶向下，将实体集分组为子类（研究生/本科生）。ISA 三角形表示。
- **概化（Generalization）**：自底向上，抽取不同实体集的相似性形成高层实体。
- 二者互逆，ER 图表示相同。
- **约束**：
  - 不相交（disjoint）/ 重叠（overlapping）
  - 全部（total）/ 部分（partial）
- **聚集（Aggregation）**：将联系抽象为更高层实体，使其能参与其他联系。

#### 5. ER 图 → 关系模式转换规则（重点）（PDF 第114页）

| ER 构造 | 转换方法 |
|---------|----------|
| 实体 | 实体 → 关系；属性 → 关系的属性 |
| 复合属性 | 将子属性作为实体的属性；复合属性本身可定义为视图 |
| 多值属性 | 新建关系 + 所在实体的主码 |
| 1:N 联系 | 将单方（1 方）实体的主码加入多方（N 方）作为外码 |
| M:N 联系 | 新建关系，属性为参与双方的主码 |
| 1:1 联系 | 若一方全部参与，将另一方主码加入全部参与方；若双方均部分参与，可视情况选择或新建关系 |
| 弱实体 | 关系主码 = 强实体主码 + 弱实体分辨符 |
| 概括（ISA） | 高层、低层各建关系；低层包含高层主码；若不相交且全部，可只建低层关系并含全部属性 |
| 聚集 | 将聚集（实体+联系）整体作为一个实体参与新联系，新联系关系含各方主码 |

### 第 3 章 关系模型

#### 1. 基本概念（PDF 第3页）
- **域（Domain）**：具有相同数据类型的一组值的集合。
- **笛卡尔积**：D₁ × D₂ × … × Dₙ = { (d₁, d₂, …, dₙ) | dᵢ ∈ Dᵢ }。
  - 元素称为 **n 元组（tuple）**，每个值 dᵢ 称为**分量（component）**。
  - 基数：|D₁| × |D₂| × … × |Dₙ|。
- **关系**：笛卡尔积的子集。R(D₁, D₂, …, Dₙ)，n 称为关系的度/目。

#### 2. 关系的性质（PDF 第14页）
1. 列同质。
2. 不同列可来自同一域，但属性名不同。
3. 行列顺序无关。
4. 任意两个元组不能完全相同。
5. 每一分量不可再分 → 满足 **1NF**。

#### 3. 关系模型三要素（PDF 第17页）
- 数据结构：单一结构——关系。
- 数据操作：选择、投影、连接、并、差等。
- 数据完整性：实体完整性、参照完整性、用户定义完整性。

#### 4. 关系模型的完整性（PDF 第29页）
- **实体完整性**：主码属性值不能为空。
- **参照完整性**：外码值要么等于被参照关系主码值，要么为空。
- **用户定义的完整性**：针对具体应用定义的约束。

#### 5. 全关系系统的十二条准则（记关键词）（PDF 第132页）
0. 完全通过关系能力管理数据库。
1. 信息准则：所有信息逻辑级用表中的值表示。
2. 保证访问准则：靠表名+主码+列名访问每个数据项。
3. 空值的系统化处理。
4. 基于关系模型的动态联机数据字典。
5. 统一的数据子语言（SQL）。
6. 视图更新准则：理论上可更新的视图系统应能更新。
7. 高级的插入、删除和修改操作（操作对象是关系）。
8. 数据物理独立性。
9. 数据逻辑独立性。
10. 数据完整性的独立性：完整性约束用数据库语言定义并存入数据字典。
11. 分布独立性。
12. 无破坏准则：低级语言不能绕过完整性准则。

#### 6. 关系代数运算（PDF 第32页）

##### 基本运算
- **选择 σ_F(R)**：在关系中选择满足条件 F 的元组。
- **投影 π_A(R)**：取若干列，结果去重。
- **更名 ρ_S(R)** / **ρ_S(A₁,…,Aₙ)(E)**：关系改名或给表达式命名。
- **笛卡尔积 R × S**：所有可能的元组组合。
- **并 R ∪ S**：相容关系（同元、对应属性域相同）。
- **差 R − S**：在 R 中但不在 S 中的元组。

##### 扩展运算
- **交 R ∩ S**：等价于 R − (R − S)。
- **自然连接 R ⋈ S**：等值连接 + 去重复列；要求同名同域属性相等。
- **θ 连接**：在笛卡尔积上按 θ 条件选择。
- **除 R ÷ S**：表示“全部包含”关系；常用于“选修了所有课程的学生”。
- **外连接**：
  - 左外连接 ⟕：自然连接 + 左侧未匹配元组。
  - 右外连接 ⟖：自然连接 + 右侧未匹配元组。
  - 全外连接 ⟗：自然连接 + 两侧未匹配元组。

##### 关系代数表达查询要点
- 多表查询先笛卡尔积/连接，再选择、投影。
- “全部”类问题常用除法。
- 同一关系自身比较需更名。

#### 7. 关系演算（能阅读即可）（PDF 第104页）
- **元组关系演算**：以元组为变量，{ t | P(t) }。
- **域关系演算**：以域为变量，如 QBE。

### 第 4 章 SQL

#### 1. SQL 特点（PDF 第8页）
- 一体化：集 DDL、DML、DCL 于一体。
- 面向集合：一次一集合。
- 高度非过程化：用户只需提出“做什么”。
- 两种使用方式：自含式（交互）和嵌入式。

#### 2. 数据类型与表的定义（了解）（PDF 第18页）
- 常见类型：
  - 整数：`int`, `smallint`, `bigint`, `tinyint`。
  - 字符串：`char(n)`（定长）, `varchar(n)`（变长）, `text`。
  - 精确小数：`decimal(p,s)` / `numeric(p,s)`（p 总位数，s 小数位）。
  - 浮点：`float`, `double`。
  - 日期时间：`date`, `time`, `datetime`, `timestamp`。
- 建表基本语法：
  ```sql
  create table 表名(
      列名 数据类型 [约束],
      ...,
      [表级约束]
  );
  ```

#### 3. 索引（PDF 第76页）
- **作用**：加快查询速度、强制唯一性、支持排序。
- **类型**：聚簇索引 / 非聚簇索引、唯一索引、复合索引、覆盖索引。
- 创建语法：
  ```sql
  create [unique] index 索引名 on 表名(列1, 列2, ...);
  ```
- **最左前缀原则**：复合索引 `(A,B,C)` 只有查询条件包含 A 或 A,B 或 A,B,C 时才能充分利用。
- 注意：索引提高查询但降低更新性能；过多索引增加存储和维护开销。

#### 4. SQL 查询基本结构（PDF 第33页）
```sql
select 目标列
from   数据源表
where  行过滤条件
[group by 分组列 [having 分组过滤条件]]
[order by 排序列]
[limit n];
```
- 等价关系代数：`π_列(σ_条件(表1 × 表2 × ...))`。

#### 5. 连接（PDF 第54页）
- **内连接**：`inner join ... on ...`
- **左外连接**：`left join ... on ...`（保留左表未匹配行）
- **右外连接**：`right join ... on ...`
- **全外连接**：`full outer join ... on ...`
- **自然连接**：`natural join`（自动按同名同值列连接）
- **笛卡尔积**：`cross join` 或 `from A, B`（不加连接条件）
- 注意：多表连接时一定要写连接条件，否则产生笛卡尔积。

#### 6. 分组与聚集（PDF 第81页）
- 聚集函数：`count`, `sum`, `avg`, `max`, `min`。
  - `count(*)` 计数所有行；`count(列)` 忽略 NULL。
  - 除 `count(*)` 外，聚集函数都忽略 NULL。
- `group by`：按指定列分组，目标列必须是分组列或聚集函数。
- `having`：对分组结果过滤，不能使用列别名。
- `with cube` / `with rollup`：生成多维汇总。

#### 7. 集合操作（PDF 第67页）
- `union [all]`：并集（默认去重）。
- `intersect [all]`：交集。
- `except [all]`：差集。
- 注意：`intersect` 优先级高于 `union`/`except`。

#### 8. 嵌套子查询（PDF 第111页）
- `in / not in`：集合成员资格。
- `some / all`：集合比较。
  - `= some` ≡ `in`；`<> all` ≡ `not in`。
- `exists / not exists`：存在性测试，常用于相关子查询。
- 复杂查询可用 `with` 定义临时视图（CTE）。

#### 9. NULL 处理（PDF 第30页）
- NULL 表示“未知”或“无意义”。
- 三值逻辑：`true`, `false`, `unknown`。
  - `and`：取 min；`or`：取 max；`not unknown = unknown`。
- 判断 NULL 用 `is null` / `is not null`，不要用 `= null`。
- 注意：两 NULL 在 `distinct`/`group by` 中被视为相等。

#### 10. 游标（服务器端脚本语言）（PDF 第13页）
- 用途：逐行处理查询结果。
- 使用步骤：
  1. `declare 游标名 cursor for select语句`
  2. `open 游标名`
  3. `fetch next from 游标名 into 变量`
  4. `close 游标名`
  5. `deallocate 游标名`

#### 11. 触发器（服务器端脚本语言）（PDF 第39页）
- 定义：事件-条件-动作规则。
- 基本语法：
  ```sql
  create trigger 触发器名
  {before|after|instead of} {insert|delete|update}
  on 表名
  [for each row]
  [when (条件)]
  begin
      动作;
  end;
  ```

### 第 5 章 完整性与安全性

#### 1. 完整性（PDF 第9页）
- **约束级别**：
  - 列级约束：列值范围。
  - 行级约束：同一行各列之间。
  - 表级约束：行间、表上、表间。
- **约束类型**：`primary key`, `foreign key`, `unique`, `not null`, `default`, `check`。
- **外键删除/更新策略**：
  - `RESTRICT`：拒绝操作。
  - `CASCADE`：级联删除/更新。
  - `SET NULL`：外码置空。
  - 语法：`foreign key (列) references 父表(列) on delete [cascade|set null|restrict] on update ...`
- **约束检查时机**：
  - 立即检查（immediate）：语句结束检查。
  - 延迟检查（deferred）：事务提交时检查。
  - 设置：`deferrable initially [deferred|immediate]`。
  - 运行时：`set constraints 约束名 deferred;`

#### 2. 安全性（PDF 第32页）
- **主体**：用户、角色、应用程序。
- **客体**：表、视图、函数、存储过程等。
- **授权**：
  ```sql
  grant 权限 on 对象 to 用户 [with grant option];
  ```
- **回收**：
  ```sql
  revoke 权限 on 对象 from 用户;
  ```
- **角色**：
  ```sql
  create role 角色名;
  grant 权限 on 对象 to 角色名;
  grant 角色名 to 用户;
  ```
- **DAC vs MAC**：
  - 自主访问控制（DAC）：主体可自主授权/回收。
  - 强制访问控制（MAC）：按许可证级别和密级，下读上写。
- **统计数据库安全**：只能查聚集值，防止通过多次查询推断个体信息。
  - 漏洞：个体太少、多次查询交叠。

### 第 6 章 多模态数据

#### 1. 递归查询（PDF 第14页）
- 用于处理层次结构、传递闭包。
- MySQL 语法：
  ```sql
  with recursive CTE as (
      -- 定位点成员（基本事实）
      select ... from BaseTable
      union all
      -- 递归成员
      select ... from CTE, BaseTable where ...
  )
  select * from CTE;
  ```

#### 2. 窗口函数（PDF 第26页）
- 语法：
  ```sql
  函数名(...) over (
      [partition by 分区列]
      [order by 排序列]
      [rows/range between ... and ...]
  )
  ```
- 常见窗口函数：
  - 聚合：`sum`, `avg`, `count`, `max`, `min` over。
  - 排名：`rank()`, `dense_rank()`, `row_number()`, `ntile(n)`。
  - 偏移：`lag()`, `lead()`。
  - 分布：`percent_rank()`, `cume_dist()`。
- `rank()`：序号重复不连续；`dense_rank()`：重复连续；`row_number()`：不重复连续。

### 第 7 章 关系规范化

#### 1. 数据异常（PDF 第5页）
- **插入异常**：该插的数据插不进去。
- **删除异常**：不该删的信息被连带删除。
- **更新异常**：修改一处需修改多处，易产生不一致。
- **数据冗余**：同一信息重复存储。

#### 2. 函数依赖（PDF 第8页）
- **定义**：R(U)，X, Y ⊆ U，若对任意关系实例，t[X]=s[X] ⇒ t[Y]=s[Y]，则 X → Y。
- **平凡函数依赖**：Y ⊆ X 时的 X → Y。
- **完全函数依赖**：X → Y，且对 X 的任意真子集 X′，X′ ↛ Y，记作 X →f Y。
- **部分函数依赖**：存在真子集 X′ → Y，记作 X →p Y。
- **传递函数依赖**：X → Y，Y → Z，Y ↛ X，Z ⊈ Y，则 X 传递决定 Z。
- **多值依赖**：X →→ Y，函数依赖是多值依赖的特例。
  - 平凡多值依赖：Y ∪ (U−X−Y) = ∅，即 Y = U−X 或 Y ⊆ X。
  - 多值依赖要求“元组产生”。

#### 3. 范式（PDF 第22页）

| 范式 | 定义 | 消除的异常 |
|------|------|-----------|
| 1NF | 属性不可再分 | 嵌套关系 |
| 2NF | 1NF 且非主属性完全函数依赖于候选码 | 非主属性对码的部分依赖 |
| 3NF | 2NF 且不存在非主属性对候选码的传递依赖 | 非主属性对码的传递依赖 |
| BCNF | 对每个非平凡函数依赖 X → Y，X 都是超码 | 主属性对码的部分/传递依赖 |
| 4NF | BCNF 且不存在非平凡多值依赖 | 非平凡多值依赖 |

- 关系：4NF ⊂ BCNF ⊂ 3NF ⊂ 2NF ⊂ 1NF。
- 一个全是主属性的关系模式一定属于 3NF。

#### 4. 逻辑蕴含与 Armstrong 公理（PDF 第91页）
- **逻辑蕴含**：F ├ X → Y 表示由 F 可推出 X → Y。
- **闭包 F⁺**：被 F 逻辑蕴涵的所有函数依赖。
- **Armstrong 公理**：
  1. 自反律：Y ⊆ X ⇒ X → Y。
  2. 增广律：X → Y ⇒ XZ → YZ。
  3. 传递律：X → Y, Y → Z ⇒ X → Z。
- 导出规则：合并律、分解律、伪传递律。
- **正确性与完备性**：Armstrong 公理导出的 ⟺ 逻辑蕴涵的。

#### 5. 属性集闭包 X⁺（PDF 第103页）
- **定义**：X⁺ = { A | X → A 可由 F 导出 }。
- **算法**：
  ```
  X⁺ := X
  while X⁺ 发生变化 do
      for each 函数依赖 A → B in F do
          if A ⊆ X⁺ then X⁺ := X⁺ ∪ B
  ```
- **用途**：
  - 判断 X → Y 是否被 F 蕴涵：Y ⊆ X⁺？
  - 求候选码：X⁺ = U 且 X 最小。

#### 6. 候选码计算（PDF 第119页）
- **属性分类**：
  - 左部属性：只出现在 F 左边 → 必在候选码中。
  - 右部属性：只出现在 F 右边 → 不在任何候选码中。
  - 外部属性：不出现在 F 中 → 必在候选码中。
  - 双向属性：左右都出现 → 可能参与。
- **求全部候选码**：从必需属性出发，逐步尝试加入双向属性，计算闭包。

#### 7. 函数依赖集等价与最小覆盖（PDF 第123页）
- **等价**：F⁺ = G⁺ ⟺ F ⊆ G⁺ 且 G ⊆ F⁺。
- **最小覆盖 Fmin** 满足：
  1. 单属性化：每个函数依赖右部为单属性。
  2. 无冗余化：去掉任意一个依赖后不等价。
  3. 既约化：每个函数依赖左部无多余属性。
- **计算步骤**：
  1. 单属性化：将 X → A₁A₂...Aₖ 拆成 X → A₁, ..., X → Aₖ。
  2. 无冗余化：对每个 X → A，令 G = F − {X → A}，若 A ∈ X⁺_G 则删除。
  3. 既约化：对 X → A，尝试去掉 X 中某属性 B，若 A ∈ (X−B)⁺_F 则用 X−B 代替 X。

#### 8. 函数依赖集在属性集上的投影（PDF 第134页）
- **定义**：π_Ui(F) = { X → Y | X → Y ∈ F⁺ 且 XY ⊆ Ui }。
- 实际计算：对 Ui 内所有属性子集 X，求 X⁺，取属于 Ui 的部分。

#### 9. 无损连接分解（PDF 第141页）
- **定义**：分解 ρ = {R₁, ..., Rₖ}，对任意实例 r，r = π_U1(r) ⋈ ... ⋈ π_Uk(r)。
- **两个子模式的无损判定**：
  - {R₁(U₁), R₂(U₂)} 无损 ⟺ U₁ ∩ U₂ → U₁ − U₂ 或 U₁ ∩ U₂ → U₂ − U₁ 成立。
- **通用判定算法（表格法）**：
  1. 构造 k 行 n 列矩阵，列是 U 中属性，行是子模式 Ui；若属性在 Ui 中则为 aⱼ，否则为 bᵢⱼ。
  2. 对每个 X → Y ∈ F：找两行在 X 上相同，则使它们在 Y 上也相同（优先改为 a）。
  3. 反复执行，若出现一行全 a，则无损；否则有损。

#### 10. 保持函数依赖分解（PDF 第169页）
- **定义**：ρ = {R₁, ..., Rₖ}，G = ∪π_Ui(F)，若 G⁺ = F⁺ 则保持函数依赖。
- **判定**：F ⊆ G⁺？即对 F 中每个 X → Y，Y ⊆ X⁺_G。

#### 11. 分解算法（PDF 第167页）
- **保持函数依赖的 3NF 分解**：
  1. 求 F 的最小覆盖 Fmin。
  2. 把 Fmin 中每个 X → A 组成一个关系模式 XA。
  3. 若所有模式都不包含候选码，则添加一个只包含候选码的关系模式。
  4. 若有模式被另一个模式包含，则去掉被包含者。
- **保持无损连接的 BCNF 分解**：
  1. 若 R 不满足 BCNF，找到违反的函数依赖 X → Y（X 不是超码）。
  2. 分解为 R₁(XY) 和 R₂(U − Y)。
  3. 递归对 R₁, R₂ 继续分解，直到都满足 BCNF。
- 注意：BCNF 分解不一定保持函数依赖；3NF 分解可保持函数依赖且保持无损连接。

### 第 8 章 事务

#### 1. 事务定义（PDF 第10页）
- 事务是数据库操作的一个逻辑工作单位，要么全做，要么全不做。
- SQL 中：`begin transaction ... commit/rollback`。

#### 2. ACID 特性（PDF 第23页）
- **原子性（Atomicity）**：事务的所有操作要么全完成，要么全不完成。
- **一致性（Consistency）**：事务执行结果必须使数据库从一个一致状态变到另一个一致状态。
- **隔离性（Isolation）**：并发事务互不干扰。
- **持久性（Durability）**：已提交事务的结果永久保存。

#### 3. 调度中的四种数据不一致性（PDF 第38页）
1. **丢失修改**：两事务同时读取并修改同一数据，后提交覆盖前者。
2. **读脏数据**：读了未提交事务修改的数据，随后该事务回滚。
3. **不可重复读**：同一事务内两次读同一数据，结果不同。
4. **幻象（Phantom）**：同一事务内两次按条件查询，结果集不同（因其他事务插入/删除）。

#### 4. 可恢复调度与无级联调度（PDF 第27页）
- **可恢复调度**：若 T₂ 读了 T₁ 写的数据，则 T₁ 必须先于 T₂ 提交。
- **无级联调度**：若 T₂ 读了 T₁ 写的数据，则 T₁ 必须在 T₂ 读之前提交。
- 关系：无级联调度 ⊂ 可恢复调度。

#### 5. SQL 四个隔离级别（PDF 第45页）

| 隔离级别 | 脏读 | 不可重复读 | 幻象 | 丢失修改 |
|----------|------|-----------|------|---------|
| Read Uncommitted | 可能 | 可能 | 可能 | 可能 |
| Read Committed | 否 | 可能 | 可能 | 可能 |
| Repeatable Read | 否 | 否 | 可能 | 否 |
| Serializable | 否 | 否 | 否 | 否 |

- 实现：通过锁的持有时间/范围控制。

#### 6. 快照隔离（Snapshot Isolation, SI）（PDF 第61页）
- 读取的是事务开始时刻（或语句开始时刻）最近已提交的数据版本。
- **SI**：事务级快照，读取事务开始时最近提交版本。
- **RCSI**：语句级快照，读取语句开始时最近提交版本。
- 可能出现写偏序（write skew）等异常。

#### 7. 冲突可串行化（PDF 第82页）
- **冲突指令**：不同事务对同一数据项的操作，且至少一个是 write。
- **冲突等价**：通过交换非冲突指令可互相转换。
- **冲突可串行化**：与一个串行调度冲突等价。
- **判定**：构造优先图，顶点为事务，边 Ti → Tj 表示 Ti 在 Tj 之前冲突。无环 ⟺ 冲突可串行化。
- 拓扑排序给出等价的串行顺序。

#### 8. 视图可串行化（PDF 第95页）
- **视图等价**：两调度中每个事务读相同初值、读相同写入值、最后写相同。
- **视图可串行化**：视图等价于某个串行调度。
- 冲突可串行化 ⟹ 视图可串行化，反之不成立。
- 判定：带标记的优先图（Tb 写初值，Tf 读终值），考虑盲目写/无用写。

### 第 9 章 并发控制

#### 1. 封锁模式及相容性（PDF 第29页）

| 现有锁 \ 请求锁 | IS | S | U | IX | SIX | X |
|----------------|----|---|---|----|-----|---|
| IS             | 是 | 是 | 是 | 是 | 是 | 否 |
| S              | 是 | 是 | 是 | 否 | 否 | 否 |
| U              | 是 | 是 | 否 | 否 | 否 | 否 |
| IX             | 是 | 否 | 否 | 是 | 否 | 否 |
| SIX            | 是 | 否 | 否 | 否 | 否 | 否 |
| X              | 否 | 否 | 否 | 否 | 否 | 否 |

- **X 锁（排它锁/写锁）**：可读可写，与其他任何锁冲突。
- **S 锁（共享锁/读锁）**：只读，与 S、IS 相容，与 X、IX、SIX、U 冲突。
- **U 锁（更新锁）**：先读后写意图，与 X 冲突，与 S、IS 相容，但本身只能有一个事务持有。
- **IS 锁（意向共享锁）**：后裔拟加 S 锁。
- **IX 锁（意向排它锁）**：后裔拟加 X 锁。
- **SIX 锁（共享意向排它锁）**：先对父节点加 S 锁，再对子节点加 IX 锁。

#### 2. 两段锁协议（2PL）（PDF 第18页）
- **增长阶段**：只能获得锁，不能释放锁。
- **缩减阶段**：只能释放锁，不能再获得锁。
- **作用**：保证冲突可串行化。
- **严格 2PL**：X 锁长锁到事务结束，防止脏读。
- **强 2PL**：所有锁长锁到事务结束。

#### 3. 死锁（PDF 第55页）
- **条件**：互斥、占有等待、非抢占、循环等待。
- **检测**：构造等待图，检测环。
- **解决**：选择牺牲者回滚。
- **预防**：
  - 破坏占有等待：一次封锁所有资源。
  - 破坏循环等待：按固定顺序加锁。
  - 破坏非抢占（按时间戳优先级，老事务优先）：
    - **wait-die**：老事务等年轻事务时等待；年轻事务等老事务时回滚（老的不死）。
    - **wound-wait**：老事务请求年轻事务持有的资源时，抢占（让年轻事务回滚）；年轻事务等老事务时等待。

#### 4. 基于时间戳的并发控制（PDF 第82页）
- 每个事务 Ti 分配时间戳 TS(Ti)，决定串行化顺序。
- 每个数据项 Q 维护：WT(Q) 最大写时间戳，RT(Q) 最大读时间戳，C(Q) 提交位。
- **规则**：
  - Ti 读 Q：若 TS(Ti) < WT(Q) → 回滚（过晚读）；否则执行读，更新 RT(Q)。
  - Ti 写 Q：若 TS(Ti) < RT(Q) 或 TS(Ti) < WT(Q) → 回滚（过晚写）；否则执行写，更新 WT(Q)。
- 提交位用于避免脏读。

#### 5. 基于有效性检查的并发控制（PDF 第91页）
- 三个阶段：读阶段 → 有效性检查阶段 → 写阶段。
- 事务三个时间戳：Start(Ti), Validation(Ti), Finish(Ti)。
- 串行化顺序按 Validation 时间戳。
- 有效性检查：比较 Ti 的读写集与其他事务的读写集，无冲突则通过，否则回滚。

#### 6. MySQL MVCC（PDF 第109页）
- 每行增加隐藏列：`db_trx_id`（最后修改事务id）、`db_roll_ptr`（undo log指针）、`db_row_id`。
- **读视图 read_view**：
  - `current_trx_id`：当前事务 id。
  - `alive_trx_list`：生成视图时活跃事务 id 列表。
  - `min_trx_id`：活跃事务最小 id。
  - `max_trx_id`：当前已创建最大事务 id + 1。
- **可见性算法**：
  - 若 `db_trx_id < min_trx_id` → 可见（已提交）。
  - 若 `db_trx_id = current_trx_id` → 可见（自己改的）。
  - 若 `db_trx_id ≥ max_trx_id` → 不可见（未来事务）。
  - 若 `min_trx_id ≤ db_trx_id < max_trx_id`：
    - 在 `alive_trx_list` 中 → 不可见（未提交）。
    - 不在 → 可见（已提交）。
  - 不可见则沿 `db_roll_ptr` 回溯 undo log 找更早版本。

### 第 10 章 恢复控制

#### 1. 故障类型（PDF 第4页）
- **事务故障**：事务未正常结束，需 undo。
- **系统故障（软故障）**：内存数据丢失，外存未破坏，如断电、DBMS 崩溃。
- **介质故障（硬故障）**：外存破坏，如磁盘损坏。

#### 2. 备份（PDF 第6页）
- **转储**：将数据库复制到磁带或磁盘。
- **静态转储**：转储期间禁止存取修改。
- **动态转储**：转储期间允许存取修改。
- **海量转储**：每次转储全部数据库。
- **增量转储**：只转储上次转储后更新过的数据。
- MySQL 备份类型：逻辑备份 / 裸文件备份；热备 / 冷备 / 温备；完全备份 / 增量备份 / 日志备份。

#### 3. 日志（PDF 第13页）
- 记录每一次更新活动：`<Ti start>`, `<Ti, X, V_old, V_new>`, `<Ti commit>`。
- **圆满事务**：日志中有 commit。
- **夭折事务**：日志中只有 start，无 commit。
- 日志类型：物理日志、逻辑日志、生理日志（physiological log）。

#### 4. WAL 原则（PDF 第25页）
- **先写日志原则（Write-Ahead Logging）**：数据页写入磁盘前，相关日志必须先写入磁盘。
- 保证：未提交修改可 undo，已提交修改可 redo。

#### 5. 基本恢复操作（PDF 第35页）
- **Redo**：对已提交事务的更新重新执行，赋予新值。幂等。
- **Undo**：对夭折事务的更新撤销，恢复旧值。幂等。
- **事务故障恢复**：反向扫描日志，undo 该事务所有更新。
- **系统故障恢复**：
  1. 正向扫描日志，将圆满事务入 redo 队列，夭折事务入 undo 队列。
  2. 反向扫描日志，对 undo 队列事务执行 undo。
  3. 正向扫描日志，对 redo 队列事务执行 redo。

#### 6. 检查点（PDF 第50页）
- **作用**：减少恢复时需要扫描的日志量。
- **模糊检查点**：生成检查点时允许事务继续运行。
- **MinLSN**：检查点起点 LSN 与最老活动事务起点 LSN 的最小值，是 redo 的起点。

#### 7. ARIES 恢复算法（PDF 第69页）
- **原则**：
  1. 先写日志（WAL）。
  2. 重做历史：重启时 redo 所有操作，包括后来回滚的。
  3. 撤销时记录补偿日志记录（CLR），支持崩溃后继续撤销。
- **重要数据结构**：
  - **脏页表 DPT**：PageID → RecLSN（使页面变脏的第一个日志 LSN）。
  - **活跃事务表 ATT**：TransID → State, LastLSN。
  - **PageLSN**：每个数据页头部记录最近更新该页的日志 LSN。
  - **PrevLSN / UndoNextLSN**：形成反向链表，支持反向 undo。
- **补偿日志记录（CLR）**：undo 某条日志时写入 CLR，记录“已经撤销了哪条 LSN”；CLR 中的 UndoNextLSN 指向下一个需要 undo 的日志，崩溃后可继续撤销而不重复。
- **三个阶段**：
  1. **分析阶段**：从最后一个检查点开始正向扫描日志，重建 DPT 和 ATT。
  2. **重做阶段**：从 DPT 中最小 RecLSN 开始正向扫描日志，对 PageLSN < LSN 的页执行 redo。
  3. **撤销阶段**：对 ATT 中未提交事务，按 LastLSN 反向 undo，并写 CLR。

### 第 11 章 数据库存储

#### 1. RAID（PDF 第22页）
- **RAID 0**：块级拆分，无冗余，高性能，低可靠性。
- **RAID 1**：磁盘镜像，写性能好，高可靠性，常用于日志文件。
- **RAID 5**：数据和奇偶校验分布到所有磁盘，读写性能较好，容单盘故障，适合读多写少。
- RAID 适用场合：
  - 日志：RAID 1/RAID 10。
  - 数据文件：RAID 5/RAID 6。

#### 2. 数据库页结构与行结构（PDF 第35页）
- **页结构**：页面头（96 字节）+ 页面体 + 页面槽（行偏移数组）。
- **行结构**：状态位 + 定长数据 + 列数 + NULL 位图 + 变长列数 + 变长列偏移数组 + 变长数据。
- **GAM**：全局分配位图，记录哪些区已分配。
- **IAM**：索引分配位图，记录对象拥有哪些区。

#### 3. B+ 树（PDF 第40页）
- 所有数据记录在叶子节点，叶子节点通过指针链接。
- 支持顺序访问和随机访问。
- 插入删除可能引起节点分裂/合并。

#### 4. LSM 树（PDF 第73页）
- 结构横跨内存和磁盘：内存中的 C0 树 + 磁盘中的 C1, C2, ... 树。
- 写操作先写内存和日志，定期合并（compaction）到磁盘。
- 适合写密集型负载。

#### 5. 位图索引（PDF 第54页）
- 每个列值对应一个位图，向量的长度等于记录数。
- 适合低基数列、分析查询，可用位运算快速回答查询。

#### 6. 按列存储（PDF 第61页）
- 每列单独存储，相同类型数据连续存放，适合分析型查询。
- 优点：高压缩率、只读取所需列、适合向量化计算。
- 缺点：插入性能差、重构元组有开销。

#### 2. 连接算法（PDF 第22页）

| 算法 | 适用条件 | 特点 |
|------|---------|------|
| 嵌套循环连接 | 任意连接条件 | 外表逐元组，内表全表扫描；小数据量可用 |
| 块嵌套循环连接 | 任意连接条件 | 按块读取；较小表作为外表 |
| 索引嵌套循环连接 | 等值/自然连接，内表有索引 | 小表驱动大表索引 |
| 归并连接 | 等值连接，输入已排序 | 顺序扫描，高效；否则需排序 |
| 哈希连接 | 等值连接 | 小表建哈希表，大表探测；适合大表无索引 |

- 连接顺序对多表查询性能影响很大。

## 三、题型专项突破

### 1. 查询题：关系代数 + SQL

#### 常见查询场景模板

**单表条件查询**
- 关系代数：`σ_条件(R)`
- SQL：`select * from R where 条件;`

**多表连接查询**
- 关系代数：`π_列(σ_连接条件(R × S))` 或 `π_列(R ⋈ S)`
- SQL：`select 列 from R join S on R.a = S.a where ...;`

**聚合查询**
- SQL：`select 分组列, avg(列) from R group by 分组列 having ...;`
- 注意：where 在分组前过滤，having 在分组后过滤。

**"至少选修了..." / "全部" 查询**
- 关系代数：用除法。
- SQL 方案 1：`not exists`：
  ```sql
  select sno from student S
  where not exists (
      select cno from course C
      where not exists (
          select * from SC
          where SC.sno = S.sno and SC.cno = C.cno
      )
  );
  ```
- SQL 方案 2：用 `count`：
  ```sql
  select sno from SC
  group by sno
  having count(distinct cno) = (select count(*) from course);
  ```

**排名 / Top-N**
- 窗口函数：
  ```sql
  select * from (
      select *, row_number() over (order by score desc) as rn
      from SC
  ) t where rn <= N;
  ```

#### 答题检查清单
- [ ] 是否写连接条件？
- [ ] 是否需要去重（distinct）？
- [ ] 聚集函数是否放在 having 而不是 where？
- [ ] NULL 是否用 is null 判断？
- [ ] 多表同名列是否加了表名前缀？

### 2. 设计题：ER 模型 + 关系规范化

#### ER 设计题步骤
1. **识别实体**：名词 → 实体。
2. **识别属性**：实体的特征，标出主码。
3. **识别联系**：动词/关联 → 联系，标出 1:1 / 1:N / M:N。
4. **处理特殊构造**：
   - 弱实体（依赖强实体，双线框 + 虚线下划线分辨符）。
   - 特化/概化（ISA 三角形）。
   - 聚集（把联系抽象为实体）。
5. **转换为关系模式**：按转换规则写出每个关系及主码、外码。

#### 规范化题步骤
1. **求候选码**：
   - 左部属性、外部属性必在候选码。
   - 右部属性不在候选码。
   - 从必需属性出发，尝试加入双向属性，求闭包 = U。
2. **判定范式**：
   - 找非主属性对候选码是否存在部分依赖 / 传递依赖。
   - 找是否存在 X → Y 且 X 不是超码。
3. **分解**：
   - 要求 BCNF + 无损连接 → BCNF 分解。
   - 要求 3NF + 保持函数依赖 + 无损连接 → 3NF 合成法。
4. **验证无损/保持函数依赖**（若题目要求）。

#### 常见易错点
- 多值属性要单独建关系。
- 1:N 联系把单方主码加入多方，不要新建关系。
- M:N 联系必须新建关系。
- 弱实体的主码 = 强实体主码 + 分辨符。
- 一个全是主属性的关系一定属于 3NF。
- BCNF 不一定保持函数依赖。

### 3. 调度分析题

#### 判断数据不一致性
1. 画出时间线，标出各事务的 read/write。
2. 看是否出现：丢失修改、读脏数据、不可重复读、幻象。

#### 判断冲突可串行化
1. 列出所有冲突指令对（不同事务、同一数据项、至少一个 write）。
2. 画优先图：若 Ti 的冲突操作在 Tj 之前，则画 Ti → Tj。
3. 检查是否有环：
   - 无环 → 冲突可串行化。
   - 有环 → 非冲突可串行化。
4. 拓扑排序得到等价串行顺序。

#### 判断视图可串行化
1. 构造带标记优先图，引入虚事务 Tb（写初值）和 Tf（读终值）。
2. 根据读来源和最后写来源加边。
3. 检查是否存在某个无环的标记优先图 → 视图可串行化。

#### 判断隔离级别能否避免不一致
- 根据课件表格：Read Uncommitted 最弱，Serializable 最强。
- 注意：Repeatable Read 只能避免脏读和不可重复读，仍可能有幻象。

#### 并发控制协议分析
- **两段锁协议**：检查事务是否先连续加锁再连续解锁；若满足，则冲突可串行化。
- **时间戳协议**：按 TS 比较 WT/RT，判断是否有过晚读/写过晚写。
- **有效性检查**：分读/验证/写三阶段，检查读写集冲突。
- **MVCC 可见性**：根据 read_view 和 db_trx_id 判断版本可见性。
